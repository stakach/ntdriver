# KmdfBasicTest KMDF Driver

Tiny KMDF `.sys` driver for the first `userspace-ntos` WDF runtime milestone.

This is intentionally **not** a KMDF DMA or interrupt driver. It is a narrow framework-runtime proof.

## Optional fake MMIO resource

This driver works without hardware resources. That lets you prove:

```text
WdfDriverCreate
EvtDeviceAdd
WdfDeviceCreate
WdfIoQueueCreate
EvtIoDeviceControl
WdfRequestCompleteWithInformation
```

before implementing `WDFCMRESLIST`.

If your PnP fixture supplies a memory resource, `EvtDevicePrepareHardware` maps it and expects:

```text
offset 0x00: ID register = 0x4b4d4446 ("KMDF")
offset 0x04: control register, optional
offset 0x08: status register, optional
```

If a memory resource exists and the ID register is not `0x4b4d4446`, `EvtDevicePrepareHardware` returns `STATUS_DEVICE_CONFIGURATION_ERROR`.

## IOCTLs

```c
IOCTL_KMDF_BASIC_PING
```

Returns:

```text
ULONG 0x4b4d4446
```

```c
IOCTL_KMDF_BASIC_ECHO
```

Copies input buffer to output buffer.

```c
IOCTL_KMDF_BASIC_GET_VERSION
```

Returns:

```c
typedef struct _KMDF_BASIC_VERSION {
    USHORT Major;
    USHORT Minor;
    USHORT Patch;
    USHORT Build;
} KMDF_BASIC_VERSION;
```

Expected value:

```text
0.1.0.1
```

```c
IOCTL_KMDF_BASIC_GET_STATE
```

Returns:

```c
typedef struct _KMDF_BASIC_STATE {
    ULONG Prepared;
    ULONG Powered;
    ULONG HasMmio;
    ULONG MmioLength;
    ULONG IoctlCount;
} KMDF_BASIC_STATE;
```

```c
IOCTL_KMDF_BASIC_READ_REG32
IOCTL_KMDF_BASIC_WRITE_REG32
```

Input/output:

```c
typedef struct _KMDF_BASIC_REG32_REQUEST {
    ULONG Offset;
    ULONG Value;
} KMDF_BASIC_REG32_REQUEST;
```

These require a mapped fake MMIO resource.

## Expected userspace-ntos flow

```text
load KmdfBasicTest.sys

DriverEntry:
  WDF_DRIVER_CONFIG_INIT
  WdfDriverCreate

WDF runtime:
  creates WDFDRIVER
  installs framework AddDevice bridge

PnP Manager:
  creates PDO/devnode
  calls AddDevice bridge

WDF runtime:
  creates WDFDEVICE_INIT
  calls EvtDriverDeviceAdd

EvtDriverDeviceAdd:
  WdfDeviceInitSetDeviceType
  WdfDeviceInitSetIoType
  WdfDeviceInitSetPnpPowerEventCallbacks
  WdfDeviceCreate
  WdfDeviceCreateSymbolicLink
  WdfIoQueueCreate

PnP Manager:
  sends START_DEVICE

WDF runtime:
  calls EvtDevicePrepareHardware
  if memory resource exists:
    WdfCmResourceListGetCount
    WdfCmResourceListGetDescriptor
    MmMapIoSpace
    READ_REGISTER_ULONG(ID)

Client:
  IOCTL_KMDF_BASIC_PING
  IOCTL_KMDF_BASIC_ECHO
  IOCTL_KMDF_BASIC_GET_VERSION
  IOCTL_KMDF_BASIC_GET_STATE

I/O Manager / WDF runtime:
  wraps IRP as WDFREQUEST
  dispatches to default queue
  calls EvtIoDeviceControl
  driver completes with WdfRequestCompleteWithInformation

Power Manager:
  D0/D3 transitions call EvtDeviceD0Entry/EvtDeviceD0Exit

PnP Manager:
  REMOVE_DEVICE calls EvtDeviceReleaseHardware and deletes WDF objects
```

## Build on Windows with WDK

Recommended:

1. Open Visual Studio with the Windows Driver Kit installed.
2. Create a new **Kernel Mode Driver, Empty (KMDF)** project.
3. Add:
   - `driver/Driver.c`
   - `driver/KmdfBasicTest.h`
4. Build x64 Release.
5. The output will be `KmdfBasicTest.sys`.

A minimal `.vcxproj` is included, but KMDF/MSBuild project files are especially WDK-version-sensitive. If it does not load, create a fresh KMDF empty project and copy the driver files in.

## Expected imports / WDF binding notes

Depending on how the WDK builds the driver, the resulting `.sys` may not directly import every `Wdf*` function. KMDF drivers often bind to the framework through:

```text
WdfVersionBind
WdfVersionUnbind
WdfFunctions function table
```

Your WDF runtime should support the binding style generated by your WDK build.

Useful functions for the first runtime slice:

```text
WdfVersionBind / WdfVersionUnbind, depending build style
WdfDriverCreate
WdfDeviceCreate
WdfDeviceCreateSymbolicLink
WdfDeviceInitSetDeviceType
WdfDeviceInitSetIoType
WdfDeviceInitSetExclusive
WdfDeviceInitSetPnpPowerEventCallbacks
WdfIoQueueCreate
WdfIoQueueGetDevice
WdfRequestRetrieveInputBuffer
WdfRequestRetrieveOutputBuffer
WdfRequestCompleteWithInformation
WdfCmResourceListGetCount
WdfCmResourceListGetDescriptor
```

WDM imports used by this test:

```text
MmMapIoSpace
MmUnmapIoSpace
READ_REGISTER_ULONG / WRITE_REGISTER_ULONG, likely macros
RtlCopyMemory / RtlZeroMemory / RtlInitUnicodeString, depending compiler
```
